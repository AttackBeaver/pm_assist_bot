# PM‑Assist Bot

**AI‑ассистент project‑менеджера для автоматизации задач из чатов и встреч**  

Бот читает текстовые и голосовые сообщения в Telegram‑чатах, распознаёт задачи, дедлайны и ответственных (включая нескольких), автоматически создаёт карточки в **YouGile**, управляет статусами, присылает уведомления и напоминания, а также предоставляет веб‑кабинет с геймификацией (XP, уровни, ачивки).

> Разработано командой **Codлета** в рамках хакатона **Цифровой прорыв 2026**

## Важные замечания

- **Автоматическое подключение к встречам через mymeet.ai** требует **корпоративного API‑ключа** (для юр. лиц). Без него эта функция недоступна, но код полностью готов к интеграции.
- **Автоматическое подключение через Playwright** требует выделенного сервера с PulseAudio и доступом к звуковому устройству. На общем хостинге (bothost.ru) не работает.
- **Веб‑кабинет на FastAPI** на bothost.ru возвращает 502 из‑за особенностей reverse‑proxy. В качестве работающей альтернативы используется **Streamlit**.
- Все остальные функции (**распознавание текста, голоса, файлов, ссылок, YouGile, геймификация, напоминания**) работают в полном объёме на любом хостинге.

## Возможности

### Автоматическое распознавание задач

| Тип | Технология |
| ----- | ----------- |
| **Текст** | Гибридный парсер: YandexGPT (основной) + regex (fallback) |
| **Голосовые сообщения** | Транскрибация через Speech2Text.ru |
| **Аудиофайлы** | MP3, WAV, OGG, AAC, WMA |
| **Видеофайлы** | WEBM, MP4, AVI, MOV, MKV (извлечение аудио через ffmpeg) |
| **Ссылки на Яндекс.Диск** | Публичные ссылки на аудио/видео файлы |
| **Несколько ответственных** | Отдельная карточка для каждого @username |
| **Дедлайны** | Естественно‑языковые выражения (сегодня, завтра, до пятницы, через 2 дня, 15 июля и др.) |

### Управление задачами в YouGile

- **Автоматическое создание** карточек в колонке «Сделать» (с дедлайном и описанием).
- **Перемещение** по колонкам: «В процессе», «Готово» (через кнопки или команды).
- **Удаление** (только автор задачи может удалить карточку).
- **Полная синхронизация** статусов с локальной БД (SQLite).

### Проактивные напоминания

- **За 2 часа до дедлайна** – личное сообщение ответственному + сообщение в групповой чат.
- **Stale‑напоминания** – раз в 6 часов о задачах, зависших более 3 дней.
- **Вечерний дайджест** – каждый день в 19:00 список активных задач каждому пользователю.

### Гибридный парсер задач

1. **Основной** – YandexGPT (облачная LLM) – высокая точность, понимает контекст.
2. **Fallback** – regex‑парсер с ключевыми словами и стоп‑словами – работает офлайн, точность >80% на тестовых фразах.
3. **Доверительный скор** – решение принимается только при confidence ≥70%.

### Обработка встреч и записей (3 способа)

#### Способ 1. Загрузка аудио/видео файла или ссылки на Яндекс.Диск

- **Работает везде** (на любом хостинге).
- Бот скачивает → транскрибирует через Speech2Text.ru → извлекает задачи → создаёт карточки в YouGile.
- Поддерживаются файлы до **20 МБ** (Telegram) и до **200 МБ** через Яндекс.Диск.

#### Способ 2. Автоматическое подключение через mymeet.ai

- **Требуется корпоративный API‑ключ mymeet.ai** (для юр. лиц).
- Пользователь отправляет ссылку на встречу → бот вызывает API mymeet.ai → сервис подключается, записывает, транскрибирует → бот получает результат и создаёт задачи.
- Поддерживаются: Яндекс Телемост, Zoom, Google Meet, Microsoft Teams, TrueConf, Jitsi и др.

#### Способ 3. Автономное подключение через Playwright + ffmpeg (собственная реализация)

- **Требуется выделенный сервер** с PulseAudio и доступом к звуковому устройству (loopback).
- Бот открывает браузер (Chromium), подключается к встрече, записывает звук через ffmpeg, транскрибирует и создаёт задачи.
- Код полностью написан, но на общем хостинге (bothost.ru) не работает из‑за отсутствия звукового устройства.

> **На демо‑хостинге** доступны способы 1 и частично 2 (при наличии API‑ключа). Для полноценного автоподключения необходим VPS с PulseAudio или корпоративный тариф mymeet.ai.

### Веб‑кабинет (два варианта)

| Вариант | Технология | Статус |
| --------- | ----------- | -------- |
| **FastAPI** | uvicorn + HTML | на bothost.ru возвращает 502 (проблема с reverse‑proxy) |
| **Streamlit** | streamlit | работает, используется как основной |

**Функционал кабинета:**

- Таблица всех задач (с фильтрацией по статусу).
- Кнопки «Выполнить», «Удалить», «Взять в работу».
- Статистика: XP, уровень, количество задач, среднее время выполнения.
- Аналитика эффективности: % выполненных в срок, среднее время в работе.
- Ачивки с иконками 🎯, ⚡, 🧙.
- Рекомендации по курсам на основе выполненных задач.

### Геймификация

- **XP** за создание задачи: +5 XP (автору).
- **XP** за выполнение задачи: +10 XP (ответственному).
- **Уровни**: 1 уровень = 100 XP.
- **Ачивки**:
  - 🎯 **Первая задача** – создана первая задача.
  - ⚡ **Спринтер** – выполнено 3 задачи.
  - 🧙‍♂️ **Мастер** – достигнут 2 уровень (200 XP).

| `/move <номер> <колонка>` | Переместить задачу |
| `/complete <номер>` | Завершить задачу |

| `👷 Тест исполнителя` | Автоматический сценарий исполнителя |

## Тестовые сценарии

- **Тест менеджера** – создание задачи → перемещение в «В процессе» → завершение → отображение XP и ачивок.
- **Тест исполнителя** – получение задачи от менеджера → взятие в работу → завершение → начисление XP.
- **Автоматический прогон** всех этапов без ручного ввода сообщений.

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <url>
cd pm_assist_bot
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv venv
source venv/bin/activate   # Linux/Mac
venv\Scripts\activate      # Windows
pip install -r requirements.txt
```

### 3. Создать файл `.env` (пример)

```env
# Telegram
BOT_TOKEN=your_telegram_bot_token

# Распознавание речи
SPEECH2TEXT_API_KEY=your_speech2text_api_key

# YouGile
YOUGILE_TOKEN=your_yougile_bearer_token
YOUGILE_BOARD_ID=your_board_id
YOUGILE_TO_COLUMN_ID=id_колонки_Сделать
YOUGILE_DO_COLUMN_ID=id_колонки_В_процессе
YOUGILE_DONE_COLUMN_ID=id_колонки_Готово

# YandexGPT (опционально, но повышает точность)
YANDEX_FOLDER_ID=your_yandex_cloud_folder_id
YANDEX_API_KEY=your_api_key

# mymeet.ai (опционально, для автоматического подключения к встречам)
MYMEET_API_KEY=your_api_key
MYMEET_API_URL=https://api.mymeet.ai/v1

# Веб‑кабинет
WEB_BASE_URL=http://localhost:8000   # или https://ваш-домен
PORT=8000
```

### 4. Запуск бота

```bash
python main.py
```

Веб‑кабинет доступен по адресу `http://localhost:8000` (FastAPI) или через отдельный процесс Streamlit.

## Docker для bothost.ru

Проект включает `Dockerfile`, который устанавливает все системные зависимости (ffmpeg, chromium, playwright). **Важно:** автоматическое подключение к встречам через Playwright требует на хосте звукового устройства (PulseAudio loopback), поэтому на bothost.ru эта функция отключена по умолчанию.

```bash
docker build -t pm-assist-bot .
docker run -d --env-file .env -p 8000:8000 pm-assist-bot
```

```bash
python qa/test_full.py
```

Проверяет: переменные окружения, парсер (≥80%), БД и геймификацию, интеграцию с YouGile, веб‑кабинет, планировщик, уведомления, YandexGPT (если настроен), Speech2Text (если есть тестовый аудиофайл).

## Примеры работы

### Текстовое сообщение в группе

```txt
@attack_beaver нужно сделать отчёт по продажам до 15 июля
```

→ Бот создаёт карточку в YouGile, @attack_beaver получает личное сообщение с кнопками управления. В чат отправляется информационное сообщение.

### Совместная задача

```txt
@ivan и @max проведите рефакторинг проекта, срок – две недели
```

→ Создаются две карточки (каждому ответственному), каждый получает уведомление.

### Обработка записи встречи

- Отправьте боту файл `.webm`, `.mp4` или аудиофайл.
- Или отправьте **публичную ссылку на Яндекс.Диск** с записью.
- Бот распознаёт речь, выделяет задачи, создаёт карточки в YouGile.

### Автоматическое подключение к встречам (при наличии условий)

- **Через mymeet.ai** – отправьте команду `/join_meet https://telemost.yandex.ru/...`
- **Через Playwright** (на сервере с PulseAudio) – отправьте команду `/join_meet_auto https://...`
