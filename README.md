# PM-Assist Bot

Telegram-бот для автоматизации управления задачами в командных чатах.  
Бот читает текстовые и голосовые сообщения, распознаёт в них задачи, дедлайны и ответственных, а затем создаёт карточки в **YouGile**.

---

## Возможности

- 📝 **Автоматическое распознавание задач** из текстовых сообщений в групповых чатах
- 🎙 **Транскрибация голосовых сообщений** через [speech2text.ru](https://speech2text.ru)
- ✅ **Создание карточек в YouGile** по подтверждению пользователя (inline-кнопки)
- ⏰ **Напоминания о дедлайнах** — за 2 часа до срока (личное сообщение + сообщение в чат)
- 📋 **Вечерний дайджест** — ежедневно в 19:00 список активных задач каждому участнику
- 🚫 **Статус «недоступен»** — команды `/away` / `/back` для управления назначением задач
- 🌐 **Веб-кабинет** (FastAPI) — просмотр, выполнение и удаление задач через браузер

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <url>
cd pm_assist_bot
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
```

### 3. Создать файл `.env`

```env
# Обязательные
BOT_TOKEN=your_telegram_bot_token
SPEECH2TEXT_API_KEY=your_speech2text_api_key

# Опциональные (без них создание задач в YouGile недоступно)
YOUGILE_TOKEN=your_yougile_bearer_token
YOUGILE_BOARD_ID=your_yougile_board_id
```

| Переменная | Где получить |
|---|---|
| `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) в Telegram |
| `SPEECH2TEXT_API_KEY` | [speech2text.ru](https://speech2text.ru) → личный кабинет |
| `YOUGILE_TOKEN` | YouGile → Настройки → API |
| `YOUGILE_BOARD_ID` | URL доски YouGile: `yougile.com/board/BOARD_ID` |

### 4. Запустить бота

```bash
python main.py
```

### 5. Запустить веб-кабинет (опционально, в отдельном терминале)

```bash
python web/app.py
```

Кабинет будет доступен по адресу:  
`http://localhost:8000/cabinet/<ваш_telegram_id>`

## Как работает распознавание задач

Бот использует эвристический парсер с системой **confidence-скора** (0–100):

| Условие | Скор |
|---|---|
| Нет ключевых слов и задача не выделена | 0 |
| Есть ключевые слова, задача не чёткая | 30 |
| Есть текст задачи (> 5 символов) | 50 |
| Задача + дедлайн **или** ответственный | 70 |
| Задача + дедлайн + ответственный | 90 |

Задача предлагается пользователю только при скоре **≥ 50**.

**Ключевые слова:** `сделать`, `задача`, `нужно`, `дедлайн`, `до`, `к`, `выполнить`, `подготовить`, `проверить`

**Примеры сообщений, которые распознаются:**
```
@ivan нужно сделать отчет до 18:00
Анна, задача: подготовить презентацию к 15.06
Нужно починить баг, дедлайн - послезавтра, возьмись max
Через 2 дня нужно сдать проект, @ivan ты ответственный
```

## Веб-кабинет

FastAPI-приложение предоставляет:

| Эндпоинт | Метод | Описание |
|---|---|---|
| `/cabinet/{telegram_id}` | GET | HTML-страница с задачами пользователя |
| `/task/{task_id}/complete` | POST | Отметить задачу выполненной |
| `/task/{task_id}/delete` | POST | Удалить задачу |
| `/health` | GET | Проверка работоспособности сервиса |

> ⚠️ Кабинет не требует аутентификации — доступ по знанию Telegram ID. Подходит для MVP.

## Известные ограничения

- **Pending-задачи в памяти** — неподтверждённые задачи хранятся в `dict` и теряются при перезапуске бота
- **Синхронные HTTP-запросы** — `Speech2TextClient` и `YouGileClient` используют синхронный `requests`, что может блокировать event loop при высокой нагрузке
- **Нет аутентификации** в веб-кабинете
- **Статус `/away`** сохраняется в БД, но пока не влияет на автоматическое назначение ответственных при парсинге
